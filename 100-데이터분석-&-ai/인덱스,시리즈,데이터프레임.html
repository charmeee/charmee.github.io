인덱스,시리즈,데이터프레임

구분 설명 구조 예시
인덱스(Index) 행(또는 열)을 식별하는 라벨(label)로, 시리즈나 데이터프레임의 각 데이터에 고유하게 할당됨 1차원 라벨 [0,2][3](https://blog.naver.com/j7youngh/222802558038) 또는 ['a','b']
시리즈(Series) 1차원 데이터 구조. 값과 인덱스(라벨)로 구성. 파이썬의 리스트나 딕셔너리와 유사하지만 인덱스가 있음 1차원 0 10
1 20
2 30
데이터프레임(DataFrame) 2차원 데이터 구조. 여러 개의 시리즈가 열(column) 단위로 결합된 형태. 행과 열 모두 인덱스(라벨)가 있음 2차원(표 형태) col1 col2
0 10 100
1 20 200

인덱스(Index)

  • 데이터의 각 행(또는 열)을 식별하는 역할을 하는 라벨입니다.
  • 시리즈와 데이터프레임 모두 인덱스를 가집니다.
  • 인덱스는 정수, 문자열, 날짜 등 다양한 형태가 될 수 있습니다.
  • 인덱스 자체는 데이터를 담지 않고, 데이터를 식별하거나 정렬, 슬라이싱 등에 사용됩니다586.

시리즈(Series)

  • 1차원 데이터 구조로, 값과 인덱스로 구성됩니다.
  • 리스트와 유사하지만, 각 값에 인덱스(라벨)가 붙어 있어 딕셔너리처럼 key-value 구조로도 볼 수 있습니다.
  • 하나의 열(column)만 존재합니다.
  • 예시: pd.Series([10,[20], index=['a', 'b', 'c'])135678.
s = pd.Series( ['4 cups', '1 cup', '2 large', '1 can'], index=['Flour', 'Milk', 'Eggs', 'Spam'], name='Dinner' ) 
df = s.to_frame()
# name부분이러면 column명이됨

데이터프레임(DataFrame)

  • 2차원 데이터 구조로, 여러 개의 시리즈(열)가 모여서 만들어집니다.
  • 행과 열 모두 인덱스를 가집니다(행 인덱스, 열 인덱스).
  • 각 열은 서로 다른 데이터 타입을 가질 수 있습니다.
  • 엑셀의 표, 데이터베이스의 테이블과 유사한 구조입니다134578.
  • df['칼람명']['인덱스명'] 
# 활용예시
fruits = pd.DataFrame({
    'Apples':[30],
    'Bananas':[21]
})
row = pd.DataFrame([[41,34]],columns=['Apples', 'Bananas'])
fruit_sales = pd.concat([fruits,row])
fruit_sales.index = ['2017 Sales','2018 Sales']
# fruit_sales['Apples']['2017 Sales'] = 35 # 되긴하는데 loc를 권장하는듯

fruit_sales.loc['2017 Sales','Apples'] = 35

정리

  • 인덱스: 데이터의 위치를 식별하는 라벨(1차원).
  • 시리즈: 값과 인덱스로 구성된 1차원 데이터(칼럼이 1개).
  • 데이터프레임: 여러 개의 시리즈(칼럼)를 모은 2차원 데이터(행과 열, 각자 인덱스)134578.

데이터프레임에서 하나의 열을 선택하면 시리즈가 되고, 시리즈와 데이터프레임 모두 인덱스를 가진다는 점이 공통점입니다.
가장 큰 차이는 시리즈는 1차원(칼럼 1개), 데이터프레임은 2차원(칼럼 여러 개)이라는 점입니다13578.
